title: Configuration system

QtRecon uses a JSON-based configuration system that stores all application settings, tool definitions, port associations, and user preferences in a single file. Configuration can be managed through both the GUI settings dialog and direct file editing.

## Configuration file location

QtRecon follows the XDG Base Directory Specification for configuration storage:

```python
# source: core/config.py:63-66
if (value := os.environ.get("XDG_CONFIG_HOME")) and (path := Path(value)).is_absolute():
    root_config_folder = path / "qtrecon"
else:
    root_config_folder = Path.home() / ".config/qtrecon"
```

**Default locations:**
- `$XDG_CONFIG_HOME/qtrecon/conf.json` (if XDG_CONFIG_HOME is set)
- `~/.config/qtrecon/conf.json` (standard Linux)

The configuration directory is created automatically on first run if it doesn't exist.

## Initial configuration setup

On first launch, QtRecon copies the example configuration:

```python
# source: core/config.py:86-92
except FileNotFoundError as e:
    if use_default_configuration:
        print(f"Default configuration file not found: {e}. Exiting.")
        sys.exit(1)
    else:
        print(f"Configuration file not found: {e}. Falling back to default example configuration...")
        Config.load_config(use_default_configuration = True)
```

The `conf.json.example` file located alongside the QtRecon binary serves as the template. This file contains:
- Core binary paths (nmap, terminal, graphical sudo)
- Default user binaries and tool configurations
- Port associations and autorun rules
- UI preferences and snippets

<Note>
If your `conf.json` becomes corrupted or unparseable, QtRecon will display an error and refuse to start. Keep a backup of your working configuration.
</Note>

## Configuration structure

The configuration file has a validated structure:

```python

conf_structure = {
```

### Default configuration values

Some settings have default values that are merged if missing:

```python
# source: core/config.py:14-17
default_configuration = {
    'user_prefs': {'monospaced_fonts': 'Hack, DejaVu Sans Mono, Droid Sans Mono, Courier, Monospace Regular'},
    'screenshots': {"engine": "qt", "interval": 15, "dst_folder": "/home/user/Images/", ...},
```

These defaults are automatically added to your configuration when the application loads, ensuring backward compatibility with older config files.

## Editing via GUI settings

The settings dialog (F10 or Options → Settings) provides a tabbed interface for configuration:

```python
# source: ui/settings.py:28-36
class Settings(QDialog):
    def __init__(self, parent):
        super(Settings, self).__init__(parent)
        self.ui = Ui_Dialog()
        self.ui.setupUi(self)
        self.config = Config.get()
        self.fill_fields_from_configuration()
```

### Settings tabs

**Tab 0: Paths**
- Nmap output directory - Where temporary scan files are stored
- Validated for write permissions before saving

```python
# source: ui/settings.py:146-156
if os.path.isdir(self.ui.nmap_output_dir.text()):
    try:
        filehandle = open(self.ui.nmap_output_dir.text() + "/write-check", 'w')
        os.remove(filehandle.name)
    except IOError:
        QMessageBox.warning(self, "Error in path configuration", 
            f"Work folder {self.ui.nmap_output_dir.text()} is not writable...")
```

**Tab 1: Core binaries**
- Nmap binary and default arguments
- Terminal emulator command
- Graphical sudo program (for privilege escalation)
- Commands are parsed with `shlex.split()` to validate syntax

**Tab 2: User binaries**
- Dynamic list of custom tools and scripts
- Each binary has: unique ID, tab title, menu text, binary path, arguments
- Searchable with real-time filtering
- Port associations (which ports show this tool)
- Autorun ports (automatically execute on these ports)

```python
# source: ui/settings.py:316-327
search_field = QLineEdit()
search_field.setClearButtonEnabled(True)
search_field.addAction(QIcon(QIcon.fromTheme(QIcon.ThemeIcon.SystemSearch)), 
    QLineEdit.LeadingPosition)
search_field.setPlaceholderText("Search...")
search_field.textChanged.connect(self.user_binaries_search_field_changed)
```

<Tip>
Use the search field in the user binaries tab to quickly find tools by name, menu text, or binary path. The list updates in real-time as you type.
</Tip>

**Tab 3: User preferences**
- Enable automatic tools (run autoruns on new services)
- Enable automatic tools on XML import
- Confirm before tab removal
- Dev null as stdin (for backgrounded processes)
- Remove nmap XML files after scan
- Delete logs on save
- Autosave interval (converted from seconds to milliseconds)
- Preferred network interfaces (for LHOST detection)
- Preferred LPORT (default listening port)
- Monospaced fonts (comma-separated list)

```python
# source: ui/settings.py:59-69
self.config["user_prefs"]['enable_autorun'] = self.ui.enable_autorun.isChecked()
self.config["user_prefs"]['autosave'] = self.ui.autosave.isChecked()
self.config["user_prefs"]['autosave_interval'] = self.ui.autosave_interval.value() * 1000
self.config["user_prefs"]['preferred_interfaces'] = self.ui.preferred_interfaces.toPlainText().split()
```

**Tab 4: Custom variables**
- User-defined variables for snippet substitution
- Format: `%%%VARIABLE_NAME%%%` in snippets gets replaced with the value
- Add/remove variables dynamically

- JSON editor for snippet definitions

### Validation and saving

When you click "OK" in the settings dialog, QtRecon:

1. **Validates mandatory fields** - Ensures binary paths, names, and critical settings are not empty
2. **Checks file permissions** - Verifies paths are writable
3. **Parses shell commands** - Uses `shlex.split()` to catch syntax errors
4. **Rebuilds port associations** - Generates autorun and ports_associations dicts from user binaries
5. **Validates JSON** - Ensures snippets are valid JSON
6. **Updates in-memory config** - Applies changes to running application
7. **Writes to disk** - Saves `conf.json` with 4-space indentation

```python
# source: ui/settings.py:39-135
def accept(self):
    if self.check_mandatory_fields(True) == False:
        return
    # ... validation and assignment logic ...
    Config.save_config()
    super(Settings, self).accept()
```

<Warning>
The settings dialog performs validation when switching tabs and on save. If validation fails, the problematic field will be highlighted in red and the tab will be focused. You cannot save until all validation errors are resolved.
</Warning>

## Reloading configuration

You can reload the configuration from disk without restarting:

**Menu:** Options → Reload configuration from file (F5)

```python
# source: core/controller.py:100-103
def reload_conf(self):
    Config.load_config(False, True)
    self.view.setup_ui()
    self.ui.statusBar().showMessage('Reloaded configuration from disk !', 5000)
```

This reloads `conf.json` and updates the UI. If the reload fails (corrupt JSON), your previous configuration remains active.

## Binary validation

On startup, QtRecon checks that all configured binaries exist and are executable:

```python
# source: core/config.py:130-138
def check_binaries() -> dict:
    ret = {'not_found': [], 'not_executable': []}
    for binary_type in ['core_binaries', 'user_binaries']:
        for binary in Config.config[binary_type]:
            if not os.path.isfile(Config.config[binary_type][binary]['binary']):
                ret['not_found'].append(Config.config[binary_type][binary]['binary'])
            elif not os.access(Config.config[binary_type][binary]['binary'], os.X_OK):
                ret['not_executable'].append(Config.config[binary_type][binary]['binary'])
    return ret
```

Missing or non-executable binaries are logged as RUNTIME warnings but don't prevent startup.

## Direct file editing

You can edit `conf.json` directly with a text editor:

1. Close QtRecon (or be prepared to reload)
2. Edit `~/.config/qtrecon/conf.json`
3. Validate your JSON syntax
4. Launch QtRecon or press F5 to reload

**Benefits:**
- Bulk editing of multiple binaries
- Version control friendly
- Easier to share configurations
- Can use comments in a `.jsonc` file, then strip them

**Risks:**
- Syntax errors will prevent startup
- No validation until load time
- Incorrect types will be rejected

<Note>
When editing user binaries, remember that port associations use the format `TCP/80` or `UDP/53`. Use `TCP/ANY` or `UDP/ANY` to match all ports of a protocol.
</Note>

## Configuration inheritance

The configuration system merges defaults with your custom config:

```python
# source: core/config.py:145-152
for first_level_key in Config.default_configuration:
    if first_level_key not in Config.config.keys():
        Config.config[first_level_key] = Config.default_configuration[first_level_key]
    else:
        for key in Config.default_configuration[first_level_key].keys():
            if key not in Config.config[first_level_key].keys():
                Config.config[first_level_key][key] = Config.default_configuration[first_level_key][key]
```

This allows upgrading QtRecon without losing your configuration - new settings get their defaults automatically.
